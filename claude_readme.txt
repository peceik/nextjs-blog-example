  NEXT.JS 블로그 예제 프로젝트 - 코드베이스 설명

프로젝트 개요
-------------
프로젝트명: my-app (Next.js 샘플 웹사이트)
버전: 0.1.0
목적: Next.js를 학습하기 위한 개인 블로그 웹사이트 스타터 템플릿
      Next.js의 사전 렌더링 개념(정적 생성 및 서버 사이드 렌더링)을
      블로그 기능을 통해 시연합니다.


기술 스택
---------
프레임워크 및 핵심:
  - Next.js v10.0.0 - 라우팅, SSR, SSG가 내장된 React 프레임워크
  - React v17.0.1 - UI 라이브러리
  - React DOM v17.0.1 - React 렌더링

콘텐츠 처리:
  - gray-matter v4.0.3 - 마크다운 파일에서 메타데이터 파싱
  - remark v13.0.0 - 마크다운 프로세서
  - remark-html v13.0.1 - 마크다운을 HTML로 변환

유틸리티:
  - date-fns v2.22.1 - 날짜 포맷팅 및 파싱

주요 파일 및 진입점
-------------------

1. 페이지 (라우트)

   /pages/_app.js
   - 모든 페이지를 감싸는 루트 컴포넌트
   - 전역 CSS를 임포트하고 적용

   /pages/index.js (홈페이지)
   - 사이트 제목, 작성자 소개, 블로그 포스트 목록 표시
   - 정적 생성(getStaticProps)을 사용하여 빌드 시점에 포스트를 가져오고 정렬
   - 주요 기능:
     * 개별 블로그 포스트로의 링크
     * 각 포스트의 날짜 포맷팅
     * 개인 소개 섹션

   /pages/posts/[id].js (동적 블로그 포스트 페이지)
   - 개별 블로그 포스트 콘텐츠 렌더링
   - getStaticPaths 및 getStaticProps와 함께 정적 생성 사용
   - 주요 기능:
     * 마크다운을 HTML로 변환
     * 포스트 제목, 날짜, 콘텐츠 표시
     * 홈으로 돌아가기 네비게이션

   /pages/api/hello.js (샘플 API 라우트)
   - JSON으로 { text: "Hello" } 반환

2. 컴포넌트

   /components/layout.js
   - 헤더와 푸터로 페이지 콘텐츠를 감쌈
   - 사이트 제목과 작성자 이름 표시 (SJ Park)
   - 프로필 이미지 표시 (원형)
   - 홈 페이지와 포스트 페이지에서 다르게 렌더링
   - SEO 메타 태그 포함 (Open Graph, Twitter Card)

   /components/date.js
   - ISO 날짜 문자열을 읽기 쉬운 형식으로 포맷 (예: "January 1, 2020")
   - date-fns 라이브러리 사용

3. 핵심 로직

   /lib/posts.js - 포스트 관리 함수:
   - getSortedPostsData()
     /posts에서 모든 마크다운 파일을 읽고 메타데이터를 파싱하여
     날짜순(최신순)으로 정렬하여 반환
   - getAllPostIds()
     정적 경로 생성을 위한 포스트 ID 배열 반환
   - getPostData(id)
     개별 포스트를 가져오고 메타데이터를 파싱하며
     마크다운을 HTML로 변환


스타일링
--------
전역 스타일 (/styles/global.css):
  - 폴백이 있는 시스템 폰트 스택
  - 링크 스타일링 (파란색, 호버 시 밑줄)
  - 이미지 최대 너비 제약
  - 18px 기본 폰트 크기, 1.6 줄 높이

CSS 모듈 (스코프 제한):
  - layout.module.css - 레이아웃 전용 스타일
  - utils.module.css - 타이포그래피 및 간격을 위한 유틸리티 클래스


사용 가능한 스크립트
--------------------
npm run dev 또는 yarn dev       # 개발 서버 시작
npm run build 또는 yarn build   # 프로덕션 빌드
npm start 또는 yarn start       # 프로덕션 서버 시작


콘텐츠 포맷
-----------
블로그 포스트는 YAML 메타데이터가 있는 마크다운 파일입니다:

---
title: '포스트 제목'
date: '2020-01-01'
---

# 여기에 마크다운 콘텐츠

포함된 예제 포스트:
  - pre-rendering.md - 정적 생성 vs 서버 사이드 렌더링 설명
  - ssg-ssr.md - 추가 Next.js 개념


아키텍처 하이라이트
--------------------

1. 정적 생성 (SSG):
   - 홈페이지와 블로그 포스트는 빌드 시점에 사전 렌더링됩니다
   - getStaticProps 및 getStaticPaths와 함께 getStaticProps 사용
   - 빠른 페이지 로드, SEO 친화적

2. 마크다운 기반 콘텐츠:
   - 블로그 포스트는 마크다운 파일로 저장
   - 자동으로 파싱되어 HTML로 변환
   - 메타데이터 (제목, 날짜)

3. 반응형 레이아웃:
   - 최적화된 이미지 로딩을 위해 Next.js Image 컴포넌트 사용
   - 컴포넌트 스코프 스타일링을 위한 CSS 모듈
   - 적절한 타이포그래피로 모바일 친화적 디자인

4. 네비게이션:
   - 홈페이지에 링크가 있는 모든 블로그 포스트 목록
   - 홈으로 돌아가기 링크가 있는 개별 포스트 페이지
   - [id].js 매개변수를 통한 동적 라우팅

시작하기
--------
1. 의존성 설치:
   npm install 또는 yarn install

2. 개발 서버 실행:
   npm run dev 또는 yarn dev

3. 브라우저에서 열기:
   http://localhost:3000

4. 블로그 포스트 편집:
   posts/ 디렉토리의 .md 파일을 수정하거나 새로 추가
